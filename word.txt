ITとは？

・IT
→　インターネットテクノロジーの略
情報技術の総称

・OS
OS
→オペレーションシステムの略
アプリやデバイスを起動させるための基本のソフトウェア

Windows
→マイクロソフト社が開発したos

macOS (Mac OS X)
→アップル社が開発したos

UNIX
→有料
現存する中でもっとも古いOS
カーナビ、企業のサーバーなどに使われている

Linux
→無料　UNIXを参考にして作られたos
オープンソース
アンドロイドのスマホなどに使われている

・言語・環境

HTML　
→Hyper Text Markup Language（ハイパーテキスト・マークアップ・ランゲージ）の略
Webページを作るための最も基本的なマークアップ言語のひとつ
WEBページの文字の事
HTMLファイルは「.html」という拡張子をつけると
HTMLファイルであることをコンピュータが認識してくれます。（例 index.html）

CSS
→Cascading Style Sheets、カスケーディング・スタイル・シートの略
ウェブページのスタイル（見栄え）（装飾、色など）を指定するための言語
HTMLと組み合わせて使う
ファイル名に拡張子の「.css」をつけることで、CSSファイルとしてコンピュータが認識する

MySQL
→マイ・エスキューエル
データベース（RDBMSの一つ）管理システムの事
データベースとは「複数で共有、利用すること」と「検索、加工すること」を
目的に整理されたデータの集まりの事。

PHP
→Hypertext Preprocessor
Web開発に特化したスクリプト言語
拡張子として「.php」
HTMLソースコードを作成することができる
MySqlやPostgerSQLなどさまざまなデータベースと連携したWebサイトを簡単に作成することができます。
チャットや掲示板、SNS、ECサイトといった動的なWebページを作成することができます。
Webページの静的なコンテンツをHTMLで書き、動的なコンテンツをPHPで記述するというすみ分けを行う。

JavaScript
→プログラミング、スクリプト言語の一つ。
JAVAとは別物
動的コンテンツを作成できる
ユーザー側のWebブラウザと、Webサイトまたはウェブサービスの相互間のやりとりを、円滑にするために使われている。
Webページで表示するHTML　CSSコンテンツを変更する事ができる

jQuery
→JavaScriptのライブラリの事
JavaScriptを使いやすいように拡張してくれているもの
JavaScriptで一から書くと何十行にもなる物が、jQueryを使用すると、ほんの数行で済む

Apache
→アパッチ
Webサーバーソフトウェア(HTTPサーバー)
Webサーバのソフトには他に「IIS（アイアイエス）」や「nginx（エンジンエックス）」などがある

Bootstrap
→レスポンシブデザインに対応したフロントエンドのCSSのフレームワーク（枠組み）の事。
WEBページでよく使われるフォーム、ボタン、メニューなどの部品がテンプレートとして用意されている。
デザイン性も確保されている

・ファイルとディレクトリ

ファイル
→パソコン内のデータのまとまり持った一つ一つのデータのこと
ファイルには、そのファイルの種類を示した拡張子（かくちょうし）という文字列が割り当ている
フォルダは複数のファイルを整理して保存するもの

ディレクトリ　
→フォルダのこと。
整理整頓され種類ごとに分類され一覧ができるもの

ルート・ディレクトリ　
→一番上のディレクトリ
最上位のディレクトリ

カレント・ディレクトリ　
→自分がいる（現在実行中の）ディレクトリ

パス
→pc内で特定の資源の所在を表す文字列のこと


絶対パス
→URLでページを指定して、目的地（情報）がどこにあるのかを確実に伝える

相対パス
→今いる場所（階層）を基準にして、目的地（情報）がどこにあるのかを伝える

・各種プロトコル

HTTP
→Hyper Text Transfer Protoco　の略
Webページを構成するHTMLファイルや、ページに関連付けられたスタイルシート、
スクリプト、画像、音声、動画などのファイルを、データ形式などのメタ情報を含めてやり取りすることができる。

FTP
→File Transfer Protocol
ファイルをネットワーク上のある場所から別の場所に移動するための基本的な手段

HTTPS
→Hypertext Transfer Protocol Secure
HTTPの内容が暗号化されているもの

SSL
→Secure Sockets Layer
インターネット上におけるウェブブラウザとウェブサーバ間でのデータの通信を暗号化し
送受信させる仕組みの

SSH
→Secure Shell
ネットワークに接続された機器を遠隔操作し、管理するための手段

ポート、ポート番号
→ネットワークでデータを通信するための扉のようなもの
ポート番号は「部屋の番号は何号室か？」という例え

HTTP、FTP、SSHの各ポート番号
→http 80
ftp 21 20
SSH 22

ファイアウォール
→企業などの内部ネットワークをインターネットを通して侵入してくる不正なアクセスから守るための“防火壁”です

TCP
→Transmission Control Protocol
IPと同様にインターネットにおいて標準的に利用されているプロトコル
ネットワーク層のIPとセッション層以上のプロトコル（例：HTTP、FTP、Telnet) の橋渡しをする形で動作しています

・IPアドレス

IPアドレス
→Internet Protocol Address
インターネット上での住所のような役割

グローバルIPアドレス
→インターネットに接続する機器が使用できるIPアドレス
プライベートIPアドレス

→プライベートIPアドレス（ローカルIPアドレス）とは、組織内の
ネットワーク（プライベートネットワーク）でのみ使用できるIPアドレスです

・ドメイン

ドメイン
→インターネットの「住所」

DNS
→Domain Name System
ドメイン名とIPアドレスの対応付けや
メールの宛先ホストを指示するためのシステムです。

VPN
→Virtual Private Network
インターネット上に仮想の専用線を設け、安全なルートを確保した上で重要な情報をやり取りすることにより
盗み見や改ざんなどの脅威から大切な情報を守ることができる。

ブリッジ
→ネットワーク中のデータ（の中にあるMACアドレス）を見て、行き先を振り分けてくれる機器のこと

プロキシ
→内部と外部のネットワークの中継をする役割を持つサーバー

・電子メール

SMTP

→メールを送信する仕組み

→Post Office Protocol

IMAP
→Internet Message Access Protocol

・セッションとクッキー
